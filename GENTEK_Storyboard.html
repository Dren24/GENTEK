Screen 3.1 Text Analyzer — Input State Module 3 · Analysis
Description

The core analysis tool embedded in the Homepage. A large textarea where the user pastes or types text. Below: a word/character counter, a Clear icon button, and the "Analyze Text" primary button. While the backend processes the request, the button shows a spinner and is disabled.

Logic
  1. User types or pastes text into the textarea; word/char counters update reactively.
  2. "Analyze Text" button becomes active when text.trim().length > 0.
  3. On click, loading = true; button disabled; spinner displayed.
  4. POST /analyze request sent with { text } payload to the FastAPI backend.
  5. NLP model classifies the text and returns { label, score, tokens[] }.
  6. On success: results saved to state; UI transitions to Screen 3.2.
  7. If user is authenticated: addToHistory(text, results) persists to DB and updates sidebar.
